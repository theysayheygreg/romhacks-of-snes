## Verified build and run lanes

### bsnes

Primary emulator/reference build on this Mac.

Build command:

```sh
cd /Users/theysayheygreg/clawd/projects/reverse-engineering-games/snes/repos/bsnes/bsnes
make -j2 platform=macos local=false
```

Verified output:

- `/Users/theysayheygreg/clawd/projects/reverse-engineering-games/snes/repos/bsnes/bsnes/out/bsnes.app`

Current caveat on this Apple Silicon Mac:

- native `arm64` `bsnes` builds currently boot ROMs with working audio but black video
- upstream has a matching open issue: `bsnes-emu/bsnes#320`
- the macOS build only exposes the `OpenGL 3.2` video backend here; there is no Metal fallback in this codebase
- the current official nightly macOS asset also resolves to an `arm64` binary, so it reproduces the same failure
- a clean `x86_64` test build was produced as a likely Rosetta workaround:
  - `/Users/theysayheygreg/Documents/SNES/emulators/bsnes-x86_64-20260309.app`
- a native Metal backend has now been implemented in the vendored source lane:
  - source branch: `codex/metal-macos`
  - source commit: `afc9e74`
  - staged test app: `/Users/theysayheygreg/Documents/SNES/emulators/bsnes-metal-20260331.app`

See also:

### SMZ3 CLI

`SMZ3` targets `net7.0`. It builds with the `.NET 9` SDK here, but launching the built binary currently requires major-version runtime roll-forward.

Build command:

```sh
source /Users/theysayheygreg/clawd/projects/reverse-engineering-games/snes/tools/macos-toolchain-env.sh
/opt/homebrew/opt/dotnet@9/bin/dotnet build /Users/theysayheygreg/clawd/projects/reverse-engineering-games/snes/repos/SMZ3/Randomizer.CLI/Randomizer.CLI.csproj -c Release
```

Verified help commands:

```sh
source /Users/theysayheygreg/clawd/projects/reverse-engineering-games/snes/tools/macos-toolchain-env.sh
/opt/homebrew/opt/dotnet@9/bin/dotnet run --project /Users/theysayheygreg/clawd/projects/reverse-engineering-games/snes/repos/SMZ3/Randomizer.CLI/Randomizer.CLI.csproj --no-build -- --help
/opt/homebrew/opt/dotnet@9/bin/dotnet run --project /Users/theysayheygreg/clawd/projects/reverse-engineering-games/snes/repos/SMZ3/Randomizer.CLI/Randomizer.CLI.csproj --no-build -- smz3 --help
```

Important caveats:

- `SMZ3` emits `net7.0` warnings because the target framework is out of support.
- Sprite-image commands in `Randomizer.CLI` use `System.Drawing.Common` and carry Windows-only warnings.
- ROM generation still expects the canonical base filenames used in `GenSeed.cs`:
  - `Super_Metroid_JU_.sfc`
  - `Zelda_no_Densetsu_-_Kamigami_no_Triforce_Japan.sfc`
